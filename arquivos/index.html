Docker - Breve Histórico

A empresa dotCloud Inc. foi fundada por Kamel Founadi, Solomon Hykes e Sebastien Pahl durante o grupo de incubadoras de startups Y Combination no verão de 2010 e lançada em 2011, e mais tarde renomeada para Docker Inc. em 2013. Hykes iniciou o projeto Docker na França como um projeto interno dentro da dotCloud, uma empresa de Plataforma como Serviço (PaaS).

Docker é uma plataforma aberta para desenvolver, entregar e executar aplicações através de containeres, permitindo que sejam separadas a aplicação da infraestrutura do projeto, permitindo assim um gerenciamento da infraestrutura tão rígido quanto das aplicações. Ao utilizar as metodologias do Docker para entrega, testagem e implantação podemos reduzir significativamente o tempo de escrita de código e execução em ambiente de produção.

O Docker está disponível para múltiplas plataformas como Linux, Windows e Mac.

Docker Compose - Visão Geral

A primeira versão dos arquivos binários da linha de comando do Docker Compose foi lançada em 2014. Foi escrita em Python, e era invocada com o comando docker-compose. Tipicamente, projetos do Compose V1 incluem um elemento de alto-nível version no arquivo compose.yml, com valores variando entre 2.0 e 3.8, que se referem a formatos de arquivos específicos. 

                    
    version: "3.8"
    services:
        db:
        image: postgres
        volumes:
            - data:/var/lib/postgresql/data
                        
    volumes:
        data:
            driver: mydriver

A segunda versão do Docker Compose foi anunciada em 2020, é escrita em Go, e é invocada com docker compose. O Compose V2 ignora o elemento version no arquivo compose.yml. 

                    
    services:
        db:
        image: postgres
        volumes:
            - data:/var/lib/postgresql/data
                        
    volumes:
        data:
            driver: mydriver
